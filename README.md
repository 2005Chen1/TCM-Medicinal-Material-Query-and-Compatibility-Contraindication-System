# 中医药常用药材查询与配伍禁忌系统



## 技术栈

- **语言**：纯Python（无第三方库）
- **数据库**：sqlite3（Python内置）
- **界面**：tkinter（Python内置）
- **数据格式**：JSON（导入导出）

## 项目结构

```
python_keshe/
├── main.py          # 程序入口：初始化DB + 启动GUI
├── database.py      # 数据库模块：sqlite3封装，所有CRUD操作
├── ui.py            # 界面模块：tkinter GUI（6个标签页）
├── data.py          # 数据模块：内置25种药材 + 17条禁忌规则
├── test.py          # 测试用例：8大测试场景，30+条断言
└── README.md        # 本文件
```

## 运行方式

```bash
# 启动桌面应用
python main.py

# 运行测试套件
python test.py
```

## 功能说明

| 标签页 | 功能 |
|--------|------|
| 药材管理 | 药材信息增删改查（名称、拼音、别名、性味、归经、功效等10个字段） |
| 药材查询 | 按名称/拼音/别名/功效分类模糊查询，点击查看详情 |
| 配伍禁忌检查 | 选择2味及以上药材，自动检测是否存在十八反/十九畏/其它禁忌 |
| 禁忌规则管理 | 添加/编辑/删除药材配伍禁忌规则 |
| 数据统计 | 药材总数、禁忌组数、功效分类分布、禁忌类型分布 |
| 数据操作 | JSON导出/导入、重置数据库（恢复内置测试数据） |

## 内置数据

- **药材**：25种常用中药材（甘草、人参、黄芪、当归、黄连、半夏、附子、川贝母、藜芦、丁香、郁金等）
- **禁忌规则**：17条配伍禁忌（覆盖十八反、十九畏核心条文）

## 自动化测试

本项目包含完整的自动化测试套件（`test.py`），共8个测试模块、40+条断言，运行命令：

```bash
python test.py
```

### 测试架构

- **测试隔离**：使用Python `tempfile` 创建临时数据库，测试结束后自动清理，不污染正式数据
- **setup/teardown**：`setup_module()` 初始化测试库并载入种子数据，`teardown_module()` 关闭连接并删除临时文件
- **结构化报告**：自定义 PASS/FAIL 断言框架，输出测试摘要与逐条详情

### 测试覆盖范围

| 模块 | 测试内容 | 断言数 | 覆盖类型 |
|------|---------|--------|---------|
| 数据库初始化 | 种子数据载入、药材/规则数量验证 | 3 | 功能测试 |
| 药材CRUD | 新增、重复拒绝、查询、更新、物理删除 | 7 | 功能测试 + 异常路径 |
| 模糊查询 | 按名称/功效/拼音查询，精确匹配 | 4 | 功能测试 |
| 配伍禁忌检查 | 十八反（甘草+甘遂、藜芦+人参）、十九畏（丁香+郁金、巴豆+牵牛子）、三味药材组合、无禁忌场景 | 8 | 规则引擎验证 + 边界条件 |
| 禁忌规则管理 | 规则增删改、重复拒绝、对称输入验证 | 5 | 功能测试 + 边界条件 |
| 数据导入导出 | JSON导出完整性（herbs/incompatibilities字段）、再导入数量一致性 | 6 | 集成测试 |
| 数据统计 | 统计字段完整性、分类非空、数量准确 | 5 | 功能测试 |
| 数据库重置 | 重置清空验证、重新导入恢复验证 | 4 | 状态恢复测试 |

### 测试设计方法

- **等价类划分**：正常药材 vs 重复名称、有效禁忌对 vs 安全对
- **边界值**：空列表、单味药材、不存在的药材名
- **错误推测**：对称输入（A+B与B+A应等价）、多味药材交叉验证
- **状态验证**：重置→清空→重新导入的数据一致性
